0-scope

Based on a circuit from here, 0-Scope is an attempt at making a low cost, cross platform oscilliscope for debugging circuits. It's a primitive tool, but for many circuits that's all you need.

I designed the board in EagleCad. It is single layer, so you can make it at home. It is meant to be milled or etched on FR-4, with a little thickness added (however you see fit).

It fits right into a USB A port, so you don't need any expensive connectors or cables.

0-scope_hardware

The AVR firmware is written using the *excellent* avr-usb libraries. All the relevant code is in C. The microcontroller runs on a PLL internal oscillator, at ~16.5MHz, and poses as a USB HID device. The scope data for each channel is relayed over HID reports.

0-scope_firmware

The host software is written in C and python. I used the libusb and libhid libraries (both cross platform), with a libhid python interface generated by swig. This let's me grab the data from the scope in a python app.

To display the data I used the matplotlib library and made two moving subplots. The software at this point is very basic, and needs a bit of work to really get full 'oscilliscope-level' functionality. But it's good for quickly and easily visualizing slow waveforms.

0-scope_software

By alternating between ADC's or overclocking a faster AVR with a higher ADC sample rate, better performance could be achieved. Even on the tiny45, by doing some software tricks, it should be possible to sacrifice the 10bit ADC performance for 8bits + a faster sample rate. In the future I would like to add some features to the software to change the time scale, measure signals, and trigger off of one probe. It would also be useful to save plots, and set certain features in the firmware (8 or 10 bit, etc).
